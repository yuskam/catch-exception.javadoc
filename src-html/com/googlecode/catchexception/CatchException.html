003    import org.mockito.cglib.proxy.MethodInterceptor;

005    import com.googlecode.catchexception.internal.DelegatingInterceptor;
006    import com.googlecode.catchexception.internal.ExceptionHolder;
007    import com.googlecode.catchexception.internal.ExceptionProcessingInterceptor;
008    import com.googlecode.catchexception.internal.InterfaceOnlyProxyFactory;
009    import com.googlecode.catchexception.internal.SubclassProxyFactory;

011    /**
012     * The methods of this class catch and verify exceptions in <em>a single line of
013     * code</em> and make them available for further analysis.
014     * 
015     * <h1>Documentation</h1>
016     * <p>
017     * <b> <a href="#1">1. How to use catch-exception?</a><br/>
018     * <a href="#2">2. What is this stuff actually good for?</a> <br/>
019     * <a href="#3">3. How does it work internally?</a><br/>
020     * <a href="#4">4. When is the caught exception reset?</a><br/>
021     * <a href="#5">5. My code throws a ClassCastException. Why?</a> <br/>
022     * <a href="#6">6. The exception is not caught. Why?</a> <br/>
023     * <a href="#7">7. Do I have to care about memory leaks?</a> <br/>
024     * <a href="#8">8. The caught exception is not available in another thread.
025     * Why?</a><br/>
026     * <a href="#9">9. How do I catch an exception thrown by a static method?</a> <br/>
027     * <a href="#10">10. Is there a way to get rid of the throws clause in my test
028     * method?</a> <br/>
029     * <a href="#11">11. Can I catch errors instead of exceptions?</a> <br/>
030     * 
031     * 
032     * 
033     * </b>
034     * <p>
035     * <h3 id="1">1. How to use catch-exception?</h3>
036     * <p>
037     * The most basic usage is:
038     * <code><pre class="prettyprint lang-java">import static com.googlecode.catchexception.CatchException.*;
039    
040    // call customerService.prepareBilling(Prize.Zero)
041    // and catch the exception if any is thrown 
042    catchException(customerService).prepareBilling(Prize.Zero);
043    
044    // assert that an IllegalArgumentException was thrown
045    assert caughtException() instanceof IllegalArgumentException;</pre></code>
046     * <p>
047     * You can combine the two lines of code in a single one if you like:
048     * <code><pre class="prettyprint lang-java">// call customerService.prepareBilling(Prize.Zero)
049    // and throw an ExceptionNotThrownAssertionError if
050    // the expected exception is not thrown 
051    verifyException(customerService, IllegalArgumentException.class).prepareBilling(Prize.Zero);</pre></code>
052     * There is a minor difference between both variants. In the first variant you
053     * must start the JVM with option <code>-ea</code> to enable the assertion. The
054     * second variant does not use JDK assertions and ,therefore, always verifies
055     * the caught exception.
056     * <p>
057     * A third variant allows you to select the type of exceptions you want to catch
058     * (no verification involved):
059     * <code><pre class="prettyprint lang-java">// catch IllegalArgumentExceptions but no other exceptions 
060    catchException(customerService, IllegalArgumentException.class).prepareBilling(Prize.Zero);</pre></code>
061     * <p>
062     * The fourth and last variant verifies that some exception is thrown, i.e. the
063     * type of the exception does not matter:
064     * <code><pre class="prettyprint lang-java">verifyException(customerService).prepareBilling(Prize.Zero);</pre></code>
065     * <p>
066     * In all variants you can use <code>caughtException()</code> afterwards to
067     * inspect the caught exception.
068     * <h3 id="2">2. What is this stuff actually good for?</h3>
069     * <p>
070     * This class targets concise and robust code in tests. Dadid Saff, a commiter
071     * to JUnit, has <a
072     * href="http://shareandenjoy.saff.net/2006/12/assertthrownexception_20.html"
073     * >discussed</a> this approach in 2007. Let me summarize the arguments here.
074     * <p>
075     * There are two advantages of the approach proposed here in comparison to the
076     * use of try/catch blocks.
077     * <ul>
078     * <li>The test is more concise and easier to read.
079     * <li>The test cannot be corrupted by a missing assertion. Assume you forgot to
080     * type <code>fail()</code> behind the method call that is expected to throw an
081     * exception.
082     * </ul>
083     * <p>
084     * There are also some advantages of this approach in comparison to test
085     * runner-specific mechanisms that catch and verify exceptions.
086     * <ul>
087     * <li>A single test can verify more than one thrown exception.
088     * <li>The test can verify the properties of the thrown exception after the
089     * exception is caught.
090     * <li>The test can specify by which method call the exception must be thrown.
091     * <li>The test does not depend on a specific test runner (JUnit4, TestNG).
092     * </ul>
093     * <p>
094     * <h3 id="3">3. How does it work internally?</h3>
095     * <p>
096     * The method <code>catchException(obj)</code> wraps the given object with a
097     * proxy that catches the exception, then (optionally) verifies the exception,
098     * and finally attaches the exception to the current <a
099     * name="threadlocal">thread</a> for further analysis. The <a
100     * href="#proxies">known limitations</a> for proxies apply.
101     * <p>
102     * Is both memory consumption and runtime a concern for you? Then use try/catch
103     * blocks instead of this class. Because in this case the creation of proxies is
104     * an unnecessary overhead. If only either memory consumption or runtime is an
105     * issue for you, feel free to configure the cache of the underlying proxy
106     * factories as appropriate.
107     * <h3 id="4">4. When is the caught exception reset?</h3>
108     * <p>
109     * The Method {@link #caughtException()} returns the exception thrown by the
110     * last method call on a proxied object in the current thread, i.e. it is reset
111     * by calling a method on the proxied object. If the called method has not
112     * thrown an exception, <code>caughtException()</code> returns null.
113     * <p>
114     * To reset the caught exception manually, call {@link #resetCaughtException()}.
115     * At the moment there is no way to reset exceptions that have been caught in
116     * other threads.
117     * <h3 id="5"><a name="proxies" />5. My code throws a ClassCastException. Why?</h3>
118     * <p>
119     * Example:
120     * <code><pre class="prettyprint lang-java">StringBuilder sb = new StringBuilder();
121    catchException(sb).charAt(-2); // throws ClassCastException</pre></code>
122     * <p>
123     * Probably you have tested a final class. Proxy factories usually try to
124     * subclass the type of the proxied object. This is not possible if the original
125     * class is final. But there is a way out. If the tested method belongs to an
126     * interface, then you can cast the argument (here: <code>sb</code>) to that
127     * interface or ,easier, change the declared type of the argument to the
128     * interface type. This works because the created proxy is not longer required
129     * to have the same type as the original class but it must only have the same
130     * interface.
131     * <code><pre class="prettyprint lang-java">// first variant
132    StringBuilder sb = new StringBuilder();
133    catchException((CharSequence) sb).charAt(-2); // works fine
134    
135    // second variant
136    CharSequence sb = new StringBuilder();
137    catchException(sb).charAt(-2); // works fine</pre></code> If the tested
138     * method does no belong to an interface fall back to the try/catch-blocks.
139     * <h3 id="6">6. The exception is not caught. Why?</h3>
140     * <p>
141     * Example:
142     * <code><pre class="prettyprint lang-java">ServiceImpl impl = new ServiceImpl();
143    catchException(impl).do(); // do() is a final method that throws an exception</pre></code>
144     * <p>
145     * Probably you have tested a final method. If that tested method belongs to an
146     * interface you could use {@link #interfaces(Object)} to fix that problem. But
147     * then the syntax starts to become ugly.
148     * <code><pre class="prettyprint lang-java">Service api = new ServiceImpl();
149    catchException(interfaces(api)).do(); // works fine</pre></code> I recommend
150     * to use try/catch blocks in such cases.
151     * 
152     * <h3 id="7">7. Do I have to care about memory leaks?</h3>
153     * <p>
154     * This library uses a {@link ThreadLocal}. ThreadLocals are known to cause
155     * memory leaks if they refer to a class the garbage collector would like to
156     * collect. If you use this library only for testing, then memory leaks do not
157     * worry you. If you use this library for other purposes than testing, you
158     * should care.
159     * <p>
160     * <h3 id="8">8. The caught exception is not available in another thread. Why?</h3>
161     * <p>
162     * The caught exception is saved <a href="#threadlocal">at the thread</a> the
163     * exception is thrown in. This is the reason the exception is not visible
164     * within any other thread.
165     * <h3 id="9">9. How do I catch an exception thrown by a static method?</h3>
166     * <p>
167     * Unfortunately, catch-exception does not support this. Fall back on try/catch
168     * blocks.
169     * <h3 id="10">10. Is there a way to get rid of the throws clause in my test
170     * method?</h3>
171     * <p>
172     * Example:
173     * <code><pre class="prettyprint lang-java">public void testSomething() throws Exception {
174        ...
175        catchException(obj).do(); // do() throws a checked exception</pre></code> No,
176     * although the exception is always caught you cannot omit the throws clause in
177     * your test method.
178     * <h3 id="11">11. Can I catch errors instead of exceptions?</h3>
179     * <p>
180     * At the moment you cannot catch {@link Throwable throwables} that are not
181     * {@link Exception exceptions}. But this could be easily changed if someone
182     * wants to get this feature.
183     * 
184     * @author rwoo
185     * @since 16.09.2011
186     */
187    public class CatchException {
188    
189        /**
190         * Returns the exception caught during the last call on the proxied object
191         * in the current thread.
192         * 
193         * @param <E>
194         *            This type parameter makes some type casts redundant.
195         * @return Returns the exception caught during the last call on the proxied
196         *         object in the current thread - if the call was made through a
197         *         proxy that has been created via
198         *         {@link #verifyException(Object, Class) verifyException()} or
199         *         {@link #catchException(Object, Class) catchException()}. Returns
200         *         null the proxy has not caught an exception. Returns null if the
201         *         caught exception belongs to a class that is no longer
202         *         {@link ClassLoader loaded}.
203         */
204        public static <E extends Exception> E caughtException() {
205            return ExceptionHolder.<E> get();
206        }
207    
208        /**
209         * Use it to verify that an exception is thrown and to get access to the
210         * thrown exception (for further verifications).
211         * <p>
212         * The following example verifies that obj.doX() throws a Exception:
213         * <code><pre class="prettyprint lang-java">verifyException(obj).doX(); // catch and verify
214    assert "foobar".equals(caughtException().getMessage()); // further analysis
215    </pre></code>
216         * <p>
217         * If <code>doX()</code> does not throw a <code>Exception</code>, then a
218         * {@link ExceptionNotThrownAssertionError} is thrown. Otherwise the thrown
219         * exception can be retrieved via {@link #caughtException()}.
220         * <p>
221         * 
222         * @param <T>
223         *            The type of the given <code>obj</code>.
224         * 
225         * @param obj
226         *            The instance that shall be proxied. Must not be
227         *            <code>null</code>.
228         * @return Returns an object that verifies that each invocation on the
229         *         underlying object throws an exception.
230         */
231        public static <T> T verifyException(T obj) {
232            return verifyException(obj, Exception.class);
233        }
234    
235        /**
236         * Use it to verify that an exception of specific type is thrown and to get
237         * access to the thrown exception (for further verifications).
238         * <p>
239         * The following example verifies that obj.doX() throws a MyException:
240         * <code><pre class="prettyprint lang-java">verifyException(obj, MyException.class).doX(); // catch and verify
241    assert "foobar".equals(caughtException().getMessage()); // further analysis
242    </pre></code>
243         * <p>
244         * If <code>doX()</code> does not throw a <code>MyException</code>, then a
245         * {@link ExceptionNotThrownAssertionError} is thrown. Otherwise the thrown
246         * exception can be retrieved via {@link #caughtException()}.
247         * <p>
248         * 
249         * @param <T>
250         *            The type of the given <code>obj</code>.
251         * 
252         * @param <E>
253         *            The type of the exception that shall be caught.
254         * @param obj
255         *            The instance that shall be proxied. Must not be
256         *            <code>null</code>.
257         * @param clazz
258         *            The type of the exception that shall be thrown by the
259         *            underlying object. Must not be <code>null</code>.
260         * @return Returns an object that verifies that each invocation on the
261         *         underlying object throws an exception of the given type.
262         */
263        public static <T, E extends Exception> T verifyException(T obj,
264                Class<E> clazz) {
265    
266            return processException(obj, clazz, true);
267        }
268    
269        /**
270         * Use it to catch an exception and to get access to the thrown exception
271         * (for further verifications).
272         * <p>
273         * In the following example you catch exceptions that are thrown by
274         * obj.doX():
275         * <code><pre class="prettyprint lang-java">catchException(obj).doX(); // catch
276    if (caughtException() != null) {
277        assert "foobar".equals(caughtException().getMessage()); // further analysis
278    }</pre></code>
279         * If <code>doX()</code> throws a exception, then {@link #caughtException()}
280         * will return the caught exception. If <code>doX()</code> does not throw a
281         * exception, then {@link #caughtException()} will return <code>null</code>.
282         * <p>
283         * 
284         * @param <T>
285         *            The type of the given <code>obj</code>.
286         * 
287         * @param obj
288         *            The instance that shall be proxied. Must not be
289         *            <code>null</code>.
290         * @return Returns a proxy for the given object. The proxy catches
291         *         exceptions of the given type when a method on the proxy is
292         *         called.
293         */
294        public static <T> T catchException(T obj) {
295    
296            return processException(obj, Exception.class, false);
297        }
298    
299        /**
300         * Use it to catch an exception of a specific type and to get access to the
301         * thrown exception (for further verifications).
302         * <p>
303         * In the following example you catch exceptions of type MyException that
304         * are thrown by obj.doX():
305         * <code><pre class="prettyprint lang-java">catchException(obj, MyException.class).doX(); // catch
306    if (caughtException() != null) {
307        assert "foobar".equals(caughtException().getMessage()); // further analysis
308    }</pre></code>
309         * If <code>doX()</code> throws a <code>MyException</code>, then
310         * {@link #caughtException()} will return the caught exception. If
311         * <code>doX()</code> does not throw a <code>MyException</code>, then
312         * {@link #caughtException()} will return <code>null</code>. If
313         * <code>doX()</code> throws an exception of another type, i.e. not a
314         * subclass but another class, then this exception is not thrown and
315         * {@link #caughtException()} will return <code>null</code>.
316         * <p>
317         * 
318         * @param <T>
319         *            The type of the given <code>obj</code>.
320         * 
321         * @param <E>
322         *            The type of the exception that shall be caught.
323         * @param obj
324         *            The instance that shall be proxied. Must not be
325         *            <code>null</code>.
326         * @param clazz
327         *            The type of the exception that shall be caught. Must not be
328         *            <code>null</code>.
329         * @return Returns a proxy for the given object. The proxy catches
330         *         exceptions of the given type when a method on the proxy is
331         *         called.
332         */
333        public static <T, E extends Exception> T catchException(T obj,
334                Class<E> clazz) {
335    
336            return processException(obj, clazz, false);
337        }
338    
339        /**
340         * Creates a proxy that processes exceptions thrown by the underlying
341         * object.
342         * <p>
343         * Delegates to
344         * {@link SubclassProxyFactory#createProxy(Class, MethodInterceptor)} which
345         * itself might delegate to
346         * {@link InterfaceOnlyProxyFactory#createProxy(Class, MethodInterceptor)}.
347         */
348        @SuppressWarnings("javadoc")
349        private static <T, E extends Exception> T processException(T obj,
350                Class<E> exceptionClazz, boolean assertException) {
351    
352            if (obj == null) {
353                throw new IllegalArgumentException("obj must not be null");
354            }
355    
356            return new SubclassProxyFactory().<T> createProxy(obj.getClass(),
357                    new ExceptionProcessingInterceptor<E>(obj, exceptionClazz,
358                            assertException));
359    
360        }
361    
362        /**
363         * Returns a proxy that implements all interfaces of the underlying object.
364         * 
365         * @param <T>
366         *            must be an interface the object implements
367         * @param obj
368         *            the object that created proxy will delegate all calls to
369         * @return Returns a proxy that implements all interfaces of the underlying
370         *         object and delegates all calls to that underlying object.
371         */
372        public static <T> T interfaces(T obj) {
373    
374            if (obj == null) {
375                throw new IllegalArgumentException("obj must not be null");
376            }
377    
378            return new InterfaceOnlyProxyFactory().<T> createProxy(obj.getClass(),
379                    new DelegatingInterceptor(obj));
380        }
381    
382        /**
383         * Sets the {@link #caughtException() caught exception} to null. This does
384         * not affect exceptions saved at threads other than the current one.
385         * <p>
386         * Actually you probably never need to call this method because each method
387         * call on a proxied object in the current thread resets the caught
388         * exception. But if you want to improve test isolation or if you want to
389         * 'clean up' after testing (to avoid memory leaks), call the method before
390         * or after testing.
391         */
392        public static void resetCaughtException() {
393            ExceptionHolder.set(null);
394        }
395
